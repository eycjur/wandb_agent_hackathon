# Cases Format

このディレクトリでは、実験ケースの正本を `inputs/` 配下の `.txt` で管理する。

理由:

- 実運用では 1 件ずつ本文を開いてアプリへコピペするため
- 評価者にそのまま配布しやすいため
- 機械処理用フォーマットとの二重管理を避けるため

## 構成

- `inputs/train/*.txt`
  - 学習用ケース 10 件のコピペ用ファイル
- `inputs/eval/*.txt`
  - 検証用ケース 4 件のコピペ用ファイル
- この `README.md`
  - ケース一覧と `assumed_primary_role` の参照先

## 運用方法

1. `inputs/train/SP-01.txt` のようなファイルを開く
2. 本文をそのままアプリに貼り付ける
3. 生成結果に対して人間評価を実施する
4. 評価記録は別途 `evaluations/` 配下のファイルへ残す

## 評価時の補足

- 原則として `assumed_primary_role` を基準に評価する
- ただし、入力文に志望職種が明示されている場合は、その志望職種を優先してよい
